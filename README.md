# Классификация токсичных комментариев (TF-IDF и BERT)

Учебный проект для сервиса модерации: по тексту комментария предсказывается, токсичный он или нет. В ноутбуке есть загрузка данных, EDA, baseline на **TF-IDF + логистическая регрессия** и вариант с эмбеддингами **BERT** и логрегом сверху.

## Задача и метрика

- **Вход:** текст комментария (`text`).
- **Выход:** бинарная метка `toxic` (0 / 1).
- **Целевая метрика:** F1 на тесте (в задании фигурирует порог **не ниже 0.75**).

Подробные выводы по данным и сравнение моделей описаны в конце ноутбука `bert-text-classifier.ipynb`.

## Данные

Ожидается CSV с колонками **`text`** и **`toxic`**. Из‑за размера файл в репозиторий обычно не включают: датасет берётся по условиям курса, `toxic_comments.csv` размещается в корне проекта рядом с ноутбуком. При другом расположении файла путь в ячейке с `pd.read_csv` меняется под фактический путь.

## Установка

Рекомендуется Python **3.10+**. Виртуальное окружение задаётся так:

```bash
cd bert-text-classifier-
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

Для **BERT** на GPU нужен PyTorch с CUDA; сборка подбирается отдельно под систему, см. [pytorch.org](https://pytorch.org). На CPU код выполняется медленнее.

## Запуск

Локально: после активации окружения ноутбук открывается в **Jupyter**, **VS Code** или **PyCharm**. В **Google Colab** ноутбук загружается через **Файл → Загрузить ноутбук**; для данных в проекте есть ячейка с `files.upload()` под Colab.

Ячейки выполняются по порядку сверху вниз. Ячейки с установкой пакетов через `pip` не обязательны, если зависимости уже установлены.

## Что внутри по моделям

- **Baseline:** `TfidfVectorizer` (униграммы и биграммы, ограничение по частоте и числу признаков) + `LogisticRegression` с `class_weight='balanced'`, разбиение train/test со стратификацией.
- **BERT:** предобученный энкодер (`bert-base-cased`), mean pooling по токенам с учётом маски, затем та же логрег по плотным векторам. Инференс идёт по батчам, чтобы снизить нагрузку на память.

Итоговые F1 и интерпретация приведены в последних разделах ноутбука (в типичном прогоне TF-IDF выше замороженного BERT без fine-tuning, что для такой постановки ожидаемо).
